## Prompt Structure

All prompts follow the **Recon-Plan-Feedback-Execute** pattern:

1. **Phase 1: RECON** - Persona: Recon/Explorer
   - Audit current state
   - Identify issues
   - **GATE: Stop and await approval**

2. **Phase 2: PLAN** - Persona: Designer/Oracle
   - Create specific fix plan
   - Define success criteria
   - **GATE: Stop and await approval**

3. **Phase 3: EXECUTE** - Persona: Fixer
   - Implement fixes
   - Verify changes
   - Commit with proper message
